CI/CD Pipeline Architecture

1 Developer Pushes Code

You run git push origin main. GitHub receives the new commits and triggers the CI workflow.

2 GitHub Actions: Install & Test

The .github/workflows/test.yml pipeline spins up Ubuntu, installs Python 3.11, runs pip install -r requirements.txt, and executes pytest app/tests/ -v.

3 Docker Build (Multi-Stage)

The Dockerfile uses a two-stage build: Stage 1 installs dependencies, Stage 2 copies only the runtime artifacts into a slim image, reducing final size by up to 60%.

4 Docker Compose Orchestration

docker-compose up starts the FastAPI app container alongside a Redis container. Environment variables connect them automatically via Docker's internal DNS.

5 Production Ready

The app serves on port 8000 with a built-in /health endpoint. Docker's HEALTHCHECK monitors it. If the process crashes, restart: unless-stopped brings it back.

Key files: Dockerfile (multi-stage image), docker-compose.yml (orchestration), .github/workflows/test.yml (CI pipeline), app/tests/test_main.py (unit tests)
